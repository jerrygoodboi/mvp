# Campus Login Scanner (PWA)

A Progressive Web App for scanning QR codes and verifying identity with biometrics (FaceID/Fingerprint) for passwordless authentication.

## Features

- **PWA**: Installable on home screen, works offline, and feels like a native app.
- **QR Scanner**: Integrated camera-based QR code scanning using `html5-qrcode`.
- **Biometrics**: Uses the Web Authentication API for secure local identity verification.
- **Secure**: Verifies sessions with the backend and stores authentication tokens.
- **Teal UI**: Clean, modern, mobile-first design.

## Prerequisites

- **HTTPS Required**: Modern browsers require HTTPS to access the camera and biometric APIs. For local testing, `localhost` is treated as secure.
- **Backend Running**: Ensure the [QR Auth Backend](../qr-auth-backend) is running on port 4000.

## Setup & Local Testing

1. **Serve the files**: Since it's a PWA, you need a local web server. You can use `npx serve`:
   ```bash
   cd qr-scanner-pwa
   npx serve .
   ```
   Or any other static server like `live-server` or Python's `http.server`.

2. **Access the app**: Open `http://localhost:3000` (or the port provided by your server) on your mobile device or desktop.

3. **Install as PWA**: On mobile (Chrome/Safari), look for "Add to Home Screen".

## How to Test

1. Enter a valid Student ID (e.g., `S12345`).
2. Click **Scan QR Code**.
3. Point your camera at a QR code generated by the [QR Auth Portal](../qr-auth-frontend).
4. Once scanned, click **Verify with Biometrics**.
5. Your device will prompt for your fingerprint or FaceID.
6. Upon success, the app sends the verification to the backend.

## Biometrics Details

The app uses `PublicKeyCredential.isUserVerifyingPlatformAuthenticatorAvailable()` to check for FaceID/TouchID/Windows Hello support. If unavailable, it falls back to a PIN-based verification.

## Deployment

To deploy this app:
1. Upload all files to a static hosting provider (Vercel, Netlify, GitHub Pages).
2. **Ensure HTTPS is enabled** for your domain.
3. Update `API_BASE_URL` in `app.js` to point to your production backend.
